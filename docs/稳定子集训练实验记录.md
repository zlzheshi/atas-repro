# ATAS 稳定子集训练实验记录

## 实验目标

在完整 ImageNet 训练前，先使用课程可控规模的 ImageNet 子集验证 ATAS 复现流程是否真实有效，而不仅是代码能够运行。

## 数据设置

- 数据集：ImageNet-1K 训练集子集
- 子集规模：100 类，每类 200 张，共 20,000 张图像
- 子集路径：`/mnt/t1b6/xuzhejia/datasets/imagenet_subset_100x200/train`
- 构造方式：从完整 ImageNet train 目录按类别抽样，并使用 symlink，避免额外复制大文件

## 关键修复

初始版本中发现 `GLD/GGD` loss 基本停在常数附近，原因有两个：

1. `batch_size=4` 时使用 `4x4` 或 `6x6` mosaic，会在一个 mosaic 内重复填充同一批图片，导致对比学习正样本目标退化。
2. 手写 OpenCLIP ViT token 提取时错误地把输入转为 `[seq, batch, dim]`，但当前 OpenCLIP Transformer 期望 batch-first 输入，导致 CLS token 几乎对所有图片相同。

已完成修复：

- 固定稳定训练配置为 `6x6` mosaic，`batch_size=36`，保证 `unique=36/36`。
- 修正 `encode_visual_tokens()`，使手写 token 路径与 OpenCLIP `encode_image()` 的 CLS 输出一致。
- 日志新增 `grid` 和 `unique` 指标，便于检查每个 mosaic 的源图像是否唯一。

## 稳定训练配置

- 配置文件：`configs/atas_vitb_subset_100x200_stable.yaml`
- 模型：OpenCLIP `ViT-B-16`
- 图像设置：
  - global image：224
  - mosaic image：960
  - mosaic grid：6x6
- 训练设置：
  - epochs：6
  - batch size：36
  - learning rate：1e-5
  - optimizer：AdamW
  - AMP：开启

第一阶段训练成功完成 3 个 epoch；在此基础上又从 `checkpoint_epoch_3.pt` 恢复训练，继续完成到 epoch 6。

- epoch 4-6：`/mnt/t1b6/xuzhejia/logs/atas_stable_resume_epoch4_6.log`

关键 loss 变化：

| 阶段 | loss | GLD | LLD | GGD | 检查项 |
| --- | ---: | ---: | ---: | ---: | --- |
| 训练开始 | 6.7590 | 3.6047 | 0.0000 | 3.1542 | `grid=6, unique=36/36` |
| epoch 3 结束 | 6.1703 | 3.2133 | 0.4432 | 2.9526 | `grid=6, unique=36/36` |
| epoch 6 结束 | 6.1139 | 3.1421 | 0.4570 | 2.9672 | `grid=6, unique=36/36` |

结果说明：

- `unique=36/36` 表明 mosaic 不再重复填充图片。
- `GGD` 从约 3.15 降到约 2.95，说明 global self-distillation 有有效学习信号。
- `GLD` 从约 3.60 降到约 3.14，说明局部区域到全局语义的对齐继续生效。
- 继续训练到 epoch 6 后，总 loss 从 epoch 3 末尾的 6.1703 降到 6.1139，但 VOC2012 简化分割指标没有同步提升，说明更长训练主要改善了当前 ImageNet 子集上的图像级表征。
- 该实验可作为课程复现中的“小规模可行性验证结果”，但还不是完整论文级 benchmark。

## checkpoint 诊断

使用 `checkpoint_epoch_3.pt` 加载后进行单 batch 诊断：

- checkpoint 加载：`missing_keys=0, unexpected_keys=0`
- 初始手写 CLS 路径已与 OpenCLIP `encode_image()` 对齐。
- 训练后同一诊断 batch 上：
  - `manual_cls_vs_manual_cls loss = 3.3572`
  - `pooled_region_vs_teacher_region loss = 3.4952`

这说明最终 checkpoint 可正常加载，且学生模型已相对初始模型发生实际更新。

后续继续训练得到的 `checkpoint_epoch_6.pt` 也已完成 kNN、VOC2012 和 patch 可视化评估，详见 `docs/ATAS_epoch6继续训练与评估结果.md`。
