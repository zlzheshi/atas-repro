# ATAS 作者设置对齐实验计划

## 目标

当前复现已经证明核心训练流程可以运行，并在 ImageNet-100x200 子集上得到更好的 kNN 表征。但和作者论文仍有明显差距：论文中 ATAS 在 VOC20 zero-shot segmentation 上从 Vanilla CLIP 的 41.8 mIoU 提升到 56.0 mIoU，而当前子集 checkpoint 在 VOC2012 简化评估上没有超过 baseline。

因此下一阶段目标是尽量靠近作者实验设置，而不是继续盲目延长子集训练。

## 作者论文中的关键设置

- 训练数据：完整 ImageNet，约 120 万张图。
- 训练轮数：6 epochs。
- 训练硬件：4 张 GPU。
- batch size：36 per GPU。
- 优化器：AdamW。
- 学习率：1e-5。
- weight decay：0.1。
- 损失权重：`GLD=1, LLD=0.01, GGD=1`。
- temperature：1。
- 下游评估：Vanilla CLIP、MaskCLIP、SCLIP、ClearCLIP、CAT-Seg、F-ViT 等 dense prediction 框架。

## 当前已接近的部分

- 完整 ImageNet 已解压到服务器：`/mnt/t1b6/xuzhejia/datasets/imagenet/train`。
- 训练代码已支持 DDP 多卡。
- 训练超参数已经和作者保持一致。
- VOC2012 baseline 接近作者 Vanilla CLIP baseline，说明评估方向大体有效。

## 下一步实验顺序

### 1. DDP smoke test

先用完整 ImageNet 跑 2 step，确认数据、DDP、显存和 mosaic 流程都正常。

配置：

```text
configs/atas_vitb_imagenet_full_author_smoke.yaml
```

状态：已完成。完整 ImageNet 数据、DDP、显存和 6x6 mosaic 流程均正常。

### 2. 完整 ImageNet 作者设置训练

使用等待脚本，在 4 张 GPU 空闲后自动启动完整训练：

```bash
nohup bash scripts/run_full_imagenet_author_wait.sh > /mnt/t1b6/xuzhejia/logs/atas_full_author_wait.log 2>&1 &
```

主配置：

```text
configs/atas_vitb_imagenet_full_author.yaml
```

输出目录：

```text
outputs/atas_vitb_imagenet_full_author/
```

状态：已启动。等待脚本已经在 4 张 GPU 空闲后进入正式训练，目前正在跑 epoch 1。

### 3. VOC checkpoint sweep

先对当前子集 epoch 1-6 全部跑 VOC2012，找出 dense proxy 最优 checkpoint：

```bash
GPU=3 bash scripts/run_voc_checkpoint_sweep.sh
```

输出目录：

```text
outputs/voc_checkpoint_sweep_subset_100x200/
```

状态：已完成。子集 epoch 1 是 ATAS checkpoint 中的最佳 VOC dense proxy，但仍低于 OpenCLIP baseline；epoch 2 到 epoch 6 呈下降趋势。因此继续延长子集训练不是最优方向。

### 4. 更接近论文的 dense evaluation

在完整 ImageNet checkpoint 产生后，优先补：

1. Vanilla VOC2012 评估。
2. MaskCLIP 风格评估。
3. SCLIP 风格评估。
4. 如果时间继续充足，再考虑 CAT-Seg 或 F-ViT。

补充：当前已有一个近似 MaskCLIP value-token 尝试，但结果明显异常，不能作为正式 MaskCLIP 对比。后续应优先接入更可靠的 MaskCLIP/SCLIP 官方或成熟实现。

## 当前判断

最接近作者结果的关键不是继续训练当前 2 万张子集，而是：

1. 使用完整 ImageNet 训练。
2. 使用多卡 batch size 36 per GPU。
3. 使用作者采用的 dense prediction 后处理/框架进行评估。
