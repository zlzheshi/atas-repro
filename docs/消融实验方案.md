# 消融实验方案

本文档记录 ATAS 复现项目的消融实验设计。目标是回答两个问题：

1. 论文中的三项蒸馏模块 `GLD`、`LLD`、`GGD` 分别对结果有什么影响。
2. 本复现中新增的 DDP `all_gather` 全局负样本是否改善训练。

## 实验控制变量

所有消融配置默认保持以下设置一致：

| 项目 | 设置 |
| --- | --- |
| Backbone | OpenAI CLIP ViT-B/16 |
| 激活函数 | QuickGELU |
| Mosaic | 6x6，960x960 |
| 优化器 | AdamW |
| 学习率 | 1e-5 |
| Weight decay | 0.1 |
| Temperature | 1.0 |
| LLD patch 采样 | `max_lld_patches=1024` |
| 默认训练步数 | 160 steps |
| 默认数据 | ImageNet-100x200 子集 |

默认使用子集和短步数，是为了快速定位模块贡献。若要做完整消融，可把 `DATA_ROOT` 指向完整 ImageNet，并移除配置中的 `max_steps`。

## 消融项

| 设置 | 配置文件 | 目的 |
| --- | --- | --- |
| `baseline` | 无 checkpoint | 原始 CLIP，不经过 ATAS 训练 |
| `full` | `configs/ablation_quickgelu_full.yaml` | GLD + LLD + GGD + all-gather |
| `no_gld` | `configs/ablation_quickgelu_no_gld.yaml` | 去掉全局到局部蒸馏，验证 GLD 是否提供局部语义迁移 |
| `no_lld` | `configs/ablation_quickgelu_no_lld.yaml` | 去掉局部到局部蒸馏，验证 LLD 是否保护 patch 结构 |
| `no_ggd` | `configs/ablation_quickgelu_no_ggd.yaml` | 去掉全局到全局蒸馏，验证 GGD 是否保护 CLS 全局语义 |
| `no_allgather` | `configs/ablation_quickgelu_no_allgather.yaml` | 保留三项损失，但只使用本地 batch 负样本 |

## 报告指标

建议报告以下指标：

| 指标 | 含义 |
| --- | --- |
| Vanilla mIoU | 直接 patch-text matching 的 VOC2012 前景 mIoU |
| SCLIP mIoU | SCLIP 风格 dense inference 的 VOC2012 前景 mIoU |
| CLS cosine | student CLS 与 teacher CLS 的平均余弦相似度 |
| Mosaic patch cosine | student mosaic patch 与 teacher mosaic patch 的平均余弦相似度 |
| Patch pairwise MSE | student/teacher patch-patch 相似度结构差异 |

## 结果表模板

| 设置 | Vanilla mIoU | SCLIP mIoU | CLS cosine | Mosaic patch cosine | Patch pairwise MSE | 结论 |
| --- | ---: | ---: | ---: | ---: | ---: | --- |
| baseline |  |  | 1.0000 | 1.0000 | 0.0000 | 原始 CLIP 参考 |
| full |  |  |  |  |  | 三项蒸馏完整版本 |
| no_gld |  |  |  |  |  | 判断 GLD 对局部语义迁移的贡献 |
| no_lld |  |  |  |  |  | 判断 LLD 对 patch 结构保护的贡献 |
| no_ggd |  |  |  |  |  | 判断 GGD 对全局语义保护的贡献 |
| no_allgather |  |  |  |  |  | 判断跨 GPU 负样本的贡献 |

## 预期解释方式

如果 `no_lld` 的 mosaic patch cosine 明显下降或 patch pairwise MSE 明显升高，说明 LLD 对局部结构保护有效。

如果 `no_ggd` 的 CLS cosine 明显下降，说明 GGD 对保留 CLIP 全局语义有效。

如果 `no_allgather` 低于 `full`，说明跨 GPU 负样本能增强 InfoNCE 约束。

如果 `full` 仍低于 baseline，则说明严格作者参数复现仍未达到论文宣称效果，应结合 drift 诊断解释失败原因。
