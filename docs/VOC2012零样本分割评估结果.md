# VOC2012 零样本分割评估结果

## 实验背景

在完成 ImageNet 子集训练、kNN 评估、消融实验和 MSD 代理分割实验后，我们进一步接入 PASCAL VOC2012，用更标准的语义分割数据集评估 ATAS checkpoint 的 patch-text 零样本 dense prediction 能力。

需要注意：当前评估不是论文中完整的下游语义分割框架，而是一个轻量级 zero-shot patch matching 评估。具体做法是提取 ViT patch token，与 20 个 VOC 类别的 CLIP 文本特征做相似度匹配，然后上采样得到像素级预测。因此它主要反映“裸 ViT patch token 与文本类别对齐”的能力。

脚本会分别评估：

## 评估指标

当前只统计 VOC 20 个前景类，不计入背景类。

- `foreground_mIoU`：20 个前景类的平均 IoU。
- `foreground_pixel_acc`：前景像素上的整体像素准确率。
- `mean_class_acc`：各前景类别准确率的平均值。

## 结果解读

从 VOC2012 结果看，Full ATAS 第 3 轮在前景 mIoU 和前景像素准确率上低于原始 OpenCLIP，但平均类别准确率略高；继续训练到第 6 轮后，ImageNet kNN 虽然提升，但 VOC2012 的 patch-text 分割指标进一步下降。具体表现为：

- ATAS 在 `car`、`horse`、`bus`、`dining table`、`boat` 等类别上有提升。
- ATAS 在 `person`、`potted plant`、`motorbike`、`train`、`bottle` 等类别上下降较明显。
- epoch 6 相比 epoch 3 在 `car` 上略有提升，但多数类别下降，说明更长训练带来的 ImageNet 子集检索提升没有直接转化为 VOC 简化分割提升。
- 说明当前 100 类 ImageNet 子集训练得到的 checkpoint 还不足以在标准 VOC zero-shot patch matching 上稳定超过原始 OpenCLIP。

这个结果并不推翻前面的 kNN 和消融实验结论，而是说明：

1. ATAS 的全局语义保持和类别间隔提升已经在 ImageNet 子集上观察到。
2. 裸 patch token 与文本类别直接匹配仍然比较粗糙。
3. 继续训练到 epoch 6 说明“更长训练”并不必然让简化分割指标提升，dense prediction 还依赖更完整的训练规模、mosaic 设置和下游分割/检测框架。
4. 当前课程复现更适合表述为“复现核心训练机制，并完成轻量级 dense proxy 验证”，而不是声称完全复现论文的最终分割精度。
