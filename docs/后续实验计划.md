# ATAS 后续实验计划

## 当前是否还需要继续实验

当前结果已经足够支撑课程复现报告：已经有核心方法实现、主实验训练、kNN 定量结果、消融实验、patch 可视化、MSD 代理分割和 VOC2012 标准数据集补充评估。

后续实验不是“必须补齐才能交”，而是用来让结论更稳、更像一篇完整复现实验报告。

## 建议优先级

### 优先级 1：整理汇报与报告

最应该先做：

1. 从 `outputs/patch_alignment_vis_epoch6_448/` 里选 2-3 张最直观的热力图。
2. 从 VOC2012 可视化里选 1-2 张成功例和 1 张失败例。
3. 把 kNN、消融、VOC2012 三张表整理进最终报告。
4. 在局限性中明确说明：epoch 6 提升了 ImageNet 子集 kNN，但没有提升 VOC2012 简化 patch-text 分割。

### 优先级 2：VOC2012 提示词和背景建模

当前 VOC2012 评估只做 20 个前景类别，不显式建模背景。后续可以尝试：

1. 加入 background / stuff / scene 等背景提示词。
2. 对每个类别使用更多 prompt template 做 prompt ensembling。
3. 调整 softmax temperature，观察 mIoU 是否敏感。
4. 对 epoch 3 和 epoch 6 同时比较，确认是否存在更适合 dense proxy 的中间 checkpoint。

这组实验成本较低，最适合继续补充。

### 优先级 3：更接近论文的 MaskCLIP/SCLIP 风格评估

论文关注 open-vocabulary dense prediction，裸 patch-text matching 比较粗糙。更进一步可以接入：

1. MaskCLIP 风格的 patch-text logits 后处理。
2. SCLIP 风格的 attention refinement。
3. 多尺度输入和滑窗推理。

这组实验更接近论文任务，但工程量会明显增加。

### 优先级 4：扩大 ImageNet 子集规模

如果服务器时间充足，可以把训练子集从 `100x200` 扩大到：

| 子集 | 图像数 | 用途 |
| --- | ---: | --- |
| 100x500 | 50,000 | 验证更多样本是否稳定提升 |
| 200x200 | 40,000 | 验证更多类别是否稳定提升 |
| 200x500 | 100,000 | 更接近正式训练，但耗时更长 |

建议先做 `200x200`，因为它能检验类间分离结论是否只依赖当前 100 个类别。

## 推荐路线

短期建议按这个顺序走：

1. 先完成 PPT 和最终报告。
2. 追加 VOC2012 prompt/background 小实验。
3. 如果还有时间，再做 `200x200` 子集训练。
4. 最后再考虑 MaskCLIP/SCLIP 风格评估。

当前最稳的论文复现结论仍然是：我们复现了 ATAS 的核心自蒸馏训练机制；在课程规模 ImageNet 子集上，Full ATAS 保持并提升了图像级语义检索能力，同时显著增大类中心间隔；但简化 dense proxy 没有完全复现论文最终分割收益。
